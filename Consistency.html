Consistency

Consistency

Consistency is used to provide consistency guarantees for read operations.

Consistency.ABSOLUTE consistency may be specified to guarantee that current values are read. Consistency.EVENTUAL consistency means that the values read may be very slightly out of date. Consistency.ABSOLUTE consistency results in higher cost, consuming twice the number of read units for the same data relative to Consistency.EVENTUAL consistency, and should only be used when required.

It is possible to set a default Consistency for a NoSQLClient instance by providing it in the initial configuration as Config#consistency. In JSON configuration file, you may use string values such as "EVENTUAL" or "ABSOLUTE". If no consistency is specified for an operation and there is no default value, Consistency.EVENTUAL is used.

Consistency can be specified in the options(opt) argument for all read operations.
